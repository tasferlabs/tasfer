<h2 id="first-doc">مستندك الأول</h2>

<Steps>
  <Step title="إنشاء مساحة">
    عند التشغيل الأول، يطلب منك تصفير إنشاء مساحة أو الانضمام إلى مساحة شخص آخر بدعوة. المساحة مجموعة من الصفحات تعيش بالكامل على جهازك في قاعدة بيانات محلية.
  </Step>
  <Step title="الكتابة">
    تعمل اختصارات Markdown أثناء الكتابة: `#` للعناوين، و`**bold**` للغامق، و`>` للاقتباس، و`- [ ]` للمهام. يُحفظ كل شيء محليًا لحظة كتابته؛ فلا يوجد زر «حفظ»، وتبقى جميع تغييراتك محفوظةً معك.
  </Step>
  <Step title="التصدير في أي وقت">
    تكتب **الإعدادات ← البيانات ← تصدير الكل** كل صفحاتك بصيغة Markdown عادية داخل ملف ZIP؛ وهي ملفات يمكنك فتحها بأي محرر اليوم أو بعد عشرين عامًا. ويمكن تصدير الصفحات منفردةً بصيغ Markdown أو HTML أو PDF أو النص العادي، كما تستورد علامة تبويب البيانات نفسها ملفات Markdown أو ZIP من جديد.
  </Step>
</Steps>

<Callout kind="note" title="لا قياس عن بُعد، مطلقًا.">
  لا يأتي تصفير مع حزمة تحليلات، ولا إرسال لتقارير الأعطال، ولا نظام حسابات.
  لا تُستخدم الشبكة إلا لغرضين: مزامنة المساحات التي شاركتها، والتحقق من تحديثات
  التطبيق على GitHub في نسخة سطح المكتب. ولا تُرفع مستنداتك إلى أي مكان. راجع
  [الخصوصية والبيانات](/docs/app/privacy) للاطلاع على التفاصيل كاملةً.

  ولأن التطبيق لا يتصل بنا خفيةً، فلن نعرف بوجود خلل إلا إذا أبلغ عنه أحد.
  واجهت مشكلة؟ [افتح بلاغًا على GitHub](https://github.com/tasferlabs/tasfer/issues).
</Callout>
