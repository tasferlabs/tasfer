<h2 id="what">ما هو؟</h2>

يوجد المُرحِّل في [`apps/live`](https://github.com/hamza512b/tasfer/tree/main/apps/live): وهو Cloudflare Worker مكتوب بـTypeScript، مع Durable Object لكل موضوع اكتشاف كي يلتقي نظراء الموضوع على النسخة نفسها. ويوفر نقطتي نهاية:

<h2 id="run">التشغيل</h2>

يتطلب النشر حساب Cloudflare وأداة Wrangler:

```bash
cd apps/live
npm install

# TURN credentials for the NAT-traversal fallback (Cloudflare Calls):
npx wrangler secret put TURN_KEY_ID
npx wrangler secret put TURN_API_TOKEN

npx wrangler dev
npx wrangler deploy
```

اضبط نطاقك في `apps/live/wrangler.toml`، ثم ابنِ التطبيق باستخدام `VITE_SIGNAL_URL=wss://signaling.example.org`. راجع [استخدام مُرحِّلك](/docs/app/sync-relay#own-relay).

<h2 id="notes">ملاحظات تشغيلية</h2>

- **تتولى Cloudflare أمر TLS والتوسع.** يعمل Worker على شبكتها الطرفية، فلا شهادات تديرها ولا موازن حمل تضبطه.
- **يمرر نصًا مشفرًا معتمًا.** تُشفّر الإشارات وإطارات المسار البديل على الجهاز قبل وصولها إلى Worker؛ فلا مفتاح فك تشفير في الخادم يحتاج إلى حمايته، وهذا هو المقصود.
- **يعتمد اليوم على Cloudflare.** يستخدم Worker كلًا من Durable Objects وCloudflare Calls لخدمة TURN، وتشغيله على بنية أخرى يتطلب نقل هذين الجزأين.
