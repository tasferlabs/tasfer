<h2 id="what">ما هو؟</h2>

يوجد المُرحِّل في [`apps/live`](https://github.com/tasferlabs/tasfer/tree/main/apps/live): وهو Cloudflare Worker مكتوب بـTypeScript، مع Durable Object لكل موضوع اكتشاف كي يلتقي نظراء الموضوع على النسخة نفسها. ويوفر نقطة نهاية واحدة:

]} />

<h2 id="run">التشغيل</h2>

يتطلب النشر حساب Cloudflare وأداة Wrangler:

```bash
cd apps/live
npm install

# TURN credentials for the NAT-traversal fallback (Cloudflare Calls):
npx wrangler secret put TURN_KEY_ID
npx wrangler secret put TURN_API_TOKEN

npx wrangler dev
npx wrangler deploy
```

اضبط نطاقك في `apps/live/wrangler.toml`، ثم ابنِ التطبيق باستخدام `VITE_SIGNAL_URL=wss://signaling.example.org`. راجع [استخدام مُرحِّلك](/docs/app/sync-relay#own-relay).

<Callout kind="note" title="ما لا يخزنه مطلقًا.">لا يحتفظ Worker بأي بيانات مستخدم. يحتفظ Durable Object لكل موضوع باتصالات WebSocket الحالية في الذاكرة؛ والبايتات الوحيدة التي يحفظها هي بيانات اعتماد TURN قصيرة العمر مخزنة مؤقتًا وعداد إصدار يومي. لا سجل رسائل ولا دليل نظراء ولا جدول حسابات — أعد تشغيله ولن يضيع لك شيء.</Callout>

<h2 id="notes">ملاحظات تشغيلية</h2>

- **تتولى Cloudflare أمر TLS والتوسع.** يعمل Worker على شبكتها الطرفية، فلا شهادات تديرها ولا موازن حمل تضبطه.
- **يمرر نصًا مشفرًا معتمًا.** تُشفّر الإشارات وإطارات المسار البديل على الجهاز قبل وصولها إلى Worker؛ فلا مفتاح فك تشفير في الخادم يحتاج إلى حمايته، وهذا هو المقصود.
- **إصدار بيانات اعتماد TURN محكوم بحدود.** لا تُصدر بيانات الاعتماد إلا لأعضاء الغرفة المتصلين — لا عبر HTTP مجهول أبدًا — ويُخزن إصدار واحد مؤقتًا لكل غرفة، وتضع ميزانية يومية إجمالية سقفًا لفاتورة TURN على الحساب في أسوأ الأحوال.
- **يعتمد اليوم على Cloudflare.** يستخدم Worker كلًا من Durable Objects وCloudflare Calls لخدمة TURN، وتشغيله على بنية أخرى يتطلب نقل هذين الجزأين.
