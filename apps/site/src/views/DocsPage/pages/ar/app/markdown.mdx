<h2 id="write">الكتابة باختصارات Markdown</h2>

ابدأ فقرة جديدة بأحد الأنماط التالية. يحوّله تصفير فور اكتمال النمط:

| اكتب | النتيجة |
| --- | --- |
| `# `, `## `, `### ` | عنوان من المستوى الأول أو الثاني أو الثالث |
| `> ` | اقتباس |
| `- `, `* `, `+ ` | قائمة نقطية |
| `1. ` | قائمة مرقّمة |
| `---` | فاصل |
| ثلاث علامات نبر خلفية | كتلة شيفرة |
| `$$` | معادلة معروضة |

ولتنسيق النص داخل السطر، ضع علامات التنسيق حول النص وأغلقها:

| اكتب | النتيجة |
| --- | --- |
| `**bold**` | غامق |
| `*italic*` | مائل |
| `~~strike~~` | يتوسطه خط |
| `` `code` `` | شيفرة داخل السطر |
| `$x^2$` | معادلة داخل السطر |

يمكنك أيضًا استخدام شريط أدوات التنسيق وقائمة `/`. تسهّل Markdown الكتابة بسرعة، وليست وضعًا يلزمك البقاء فيه.

<h2 id="blocks">ما يدعمه Markdown</h2>

يقرأ تصفير العناصر التالية ويكتبها:

- الفقرات والعناوين من المستويات 1–3
- النص الغامق والمائل والمشطوب، والشيفرة داخل السطر، والروابط
- الاقتباسات والفواصل والقوائم النقطية والمرقّمة وقوائم المهام
- القوائم المتداخلة، مع استخدام مسافتين لكل مستوى من المسافة البادئة في Markdown المُصدّر
- كتل الشيفرة المسيّجة، مع إمكانية تحديد لغة لتمييز الصياغة
- الصور بصيغة `![alt](url)`
- LaTeX داخل السطر بين `$...$`، وLaTeX المعروض بين `$$...$$`

<Callout kind="note" title="تصفير ليس محررًا لنص Markdown الخام.">
  يعرض المحرر محتوى منظمًا ومنسقًا بدلًا من لوحة تعرض النص المصدري. Markdown
  هي صيغة الإدخال والإخراج القابلة للنقل في تصفير، أما المستند أثناء العمل
  فيُخزن كحالة CRDT حتى تندمج التعديلات بين الأجهزة بأمان.
</Callout>

<h2 id="import-export">الاستيراد والتصدير</h2>

استورد ملفات `.md` من **الإعدادات ← البيانات ← استيراد**. يمكنك استيراد ملفات Markdown منفردة أو ملف ZIP يحتوي عليها.

صدّر صفحة واحدة بصيغة Markdown من قائمة الصفحة. ولنسخ جميع الصفحات احتياطيًا دفعة واحدة، استخدم **الإعدادات ← البيانات ← تصدير الكل**؛ وسيُنشئ تصفير ملف ZIP يضم ملفات `.md` عادية. ويمكنك من قائمة الصفحة نفسها التصدير بصيغ HTML أو PDF أو النص العادي.

<Callout kind="tip" title="احتفظ بنسخة قابلة للنقل.">
  تصدير Markdown هو نسخة مستقلة يمكنك فتحها في محرر آخر، أو حفظها في Git،
  أو استخدامها كنسخة احتياطية. وتظل قاعدة بيانات تصفير المحلية هي مصدر
  الحقيقة للتحرير والمزامنة من نظير إلى نظير.
</Callout>
