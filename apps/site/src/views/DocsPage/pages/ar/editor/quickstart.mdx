<h2 id="value">1 — إدخال القيمة</h2>

يمكن إنشاء المحرر من Markdown أو مستند CRDT أو من دون قيمة. وهذه نقاط بداية متكافئة:

<FrameworkTabs>

```ts
import { createEditor, createDoc } from "@tasfer/editor";

createEditor({ element, markdown: "# Title" });

const doc = createDoc(savedBytes);
createEditor({ element, doc });

createEditor({ element });
```

```tsx
import { Editor, useEditor } from "@tasfer/react";
import { createDoc } from "@tasfer/editor";

<Editor markdown="# Title" />;

const doc = createDoc(savedBytes);
<Editor doc={doc} />;

<Editor />;

const { containerRef, editor } = useEditor({ markdown: "# Title" });
```

</FrameworkTabs>

<h2 id="state">2 — قراءة الحالة</h2>

اقرأ التحديد والعلامات النشطة من لقطة غير قابلة للتغيير: مباشرة عبر `editor.state`، أو عبر `useEditorState(editor)` في React. يعيد الخطاف البنية نفسها ويعيد عرض المكوّن عند تغيرها. لا تتغير اللقطة من تحتك؛ بل ينتج كل تعديل لقطة جديدة.

<FrameworkTabs>

```ts
const { selection, activeMarks, doc } = editor.state;

selection.empty;
activeMarks.has("strong");
```

```tsx
import { useEditorState } from "@tasfer/react";

const state = useEditorState(editor);
const active = state?.activeMarks ?? new Set();

state?.selection.empty;
active.has("strong");
```

</FrameworkTabs>

<h2 id="commands">3 — إجراء التغييرات</h2>

لا تغيّر المستند مباشرة. استدعِ `change(fn)` لتحصل الدالة على `ChangeApi`. تصف كل دالة تعديلًا داخل معاملة — يُطبق على CRDT أو لا يفعل شيئًا — ويمكن تسلسلها. تلتزم الاستدعاءات كلها في **خطوة تراجع واحدة**، ويعيد `change` ما إذا تغير شيء.

```ts
editor.change((c) => c.setMark("strong"));
editor.change((c) => c.setBlock({ type: "heading", level: 2 }));

editor.change((c) =>
  c.setMark("strong").setMark("emphasis").insertText(" ✶"),
);
```

التغييرات مستقلة عن إطار العمل. في React تستدعي `change` على المقبض نفسه من `useEditor` أو `onReady`، بما في ذلك داخل `onClick`. لا توجد واجهة تغييرات منفصلة لـReact.

تجد `ChangeApi` كاملًا والتجارب الجافة عبر `canChange` وحافلة الأفعال في [التغييرات والأفعال](/docs/editor/api-commands).

<h2 id="events">4 — الأحداث</h2>

اشترك فيما يفعله المحرر. يعيد كل مستمع دالة لإلغاء الاشتراك.

<PropsTable cols={["الحدث", "وقت الإطلاق", "الحمولة"]} rows={[
  { name: "change", type: "ChangeTransaction", desc: "تغير المستند بالكتابة أو اللصق أو التراجع أو مزامنة بعيدة. الحمولة: { isRemote, ops }." },
  { name: "selectionchange", type: "EditorStateSnapshot", desc: "تحرك المؤشر أو التحديد من دون تعديل المستند." },
  { name: "focus / blur", type: "EditorStateSnapshot", desc: "اكتسبت اللوحة التركيز أو فقدته." },
]} />

لا تعد حالة الاتصال وعدد النظراء من أحداث المحرر؛ فهما على المزود عبر `provider.on("sync")` لأن المزود يملك النقل. راجع [التعاون](/docs/editor/collaboration).

<FrameworkTabs>

```ts
const off = editor.on("change", (tx) => {
  if (tx.isRemote) return;
  console.log("local edit:", tx.ops.length, "ops");
});
off();
```

```tsx
useEffect(() => {
  if (!editor) return;
  return editor.on("change", (tx) => {
    if (tx.isRemote) return;
    console.log("local edit:", tx.ops.length, "ops");
  });
}, [editor]);
```

</FrameworkTabs>
