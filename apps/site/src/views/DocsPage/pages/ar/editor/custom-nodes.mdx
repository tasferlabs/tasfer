<h2 id="node">تعريف عقدة</h2>

العقدة نوع من الكتل. وهي تحدد سماتها (حقول CRDT المنسوخة)، وكيفية تسلسلها ورسمها. إليك صندوق تنبيه:

```ts
import { defineNode } from "@tasfer/editor";

const callout = defineNode("callout", {
  // Replicated attributes — each becomes a CRDT-settable field with a default.
  attrs: { tone: { default: "note" } },
  // Declarative paint for the built-in box renderer (or pass `node` for a
  // custom canvas Node). Round-trips through a generic <x-callout tone="…" />
  // tag, so no tokenizer changes are needed.
  render: {
    background: "rgba(29,185,132,0.10)",
    borderLeft: { width: 3, color: "#1db984" },
    label: (block) => `Callout (${block.tone})`,
  },
});
```

أتفضّل استخدام صنف؟ أنشئ صنفًا فرعيًا من عقدة وسجّل مثيله مباشرةً. يملك الصنف رسمه وسلاسله وواجهته، ويوفر `static nodeConfig` السمات والتسلسل نفسيهما اللذين يستقبلهما `defineNode`. ينتج الأسلوبان بيانات متطابقة، لذا يمكن استخدام أحدهما مكان الآخر:

```ts
import { AtomicNode, baseSchema } from "@tasfer/editor";

class CalloutNode extends AtomicNode {
  readonly type = "callout";
  // Attrs + serialization, read when the class is registered directly.
  static nodeConfig = { attrs: { tone: { default: "note" } } };

  protected intrinsicHeight() {
    return 48;
  }
  protected draw(box, c) {
    c.ctx.fillStyle = "rgba(29,185,132,0.10)";
    c.ctx.fillRect(box.x, box.y, box.width, 48);
  }
}

const schema = baseSchema.extend({ nodes: [new CalloutNode()] });
```

<h2 id="strings">السلاسل المترجمة مكانها العقدة</h2>

تملك العقدة السلاسل التي ترسمها على لوحة الرسم، مثل تسميات الحالة وعبارات التوجيه، في كتالوج `strings` ذي قيم إنجليزية افتراضية. لا يأتي المحرر مع مكتبة تدويل؛ بل يتجاوز المضيف المعرّب القيم لكل نوع عبر `theme.nodeStrings`. اقرأها داخل `draw` باستخدام الدالة المحمية المساعدة `str(state, key)`. (تملك عقدة الصور الأساسية وعقدة الرياضيات الاختيارية تسمياتهما بالطريقة نفسها تمامًا؛ فلم تعد موجودة في جدول سلاسل عام.)

```ts
class CalloutNode extends AtomicNode {
  readonly type = "callout";
  readonly strings = { hint: "Add a note…" };

  protected draw(box, c) {
    c.ctx.fillText(this.str(c.state, "hint"), box.x + 12, box.y + 24);
  }
}

// Host overrides, keyed by block type then the node's local string key:
createEditor({
  element,
  schema,
  theme: { nodeStrings: { callout: { hint: "ملاحظة…" } } },
});
```

<h2 id="overlays">واجهة تملكها العقدة (مواضع التراكب)</h2>

يمكن للعقدة أن تطلب من المضيف إظهار عناصر واجهة فعلية عائمة، مثل نافذة منبثقة أو محرر مضمّن، فوق إحدى كتلها من دون أن يستورد المحرك إطار عمل الواجهة. تعيد `overlays()` *واصفات* مستقلة عن العارض (الهوية والهندسة المشتقتان من بيانات الكتلة وحالة الواجهة الحالية)؛ ويربط المضيف كل `key` بمكوّن ويثبّته عند `rect` الخاص بالواصف.

```ts
class CalloutNode extends AtomicNode {
  readonly type = "callout";
  overlays(c) {
    // Show an editor overlay while this block holds the caret.
    if (c.state.document.cursor?.position.blockIndex !== c.blockIndex)
      return [];
    return [
      {
        key: "callout-editor",
        blockIndex: c.blockIndex,
        rect: { x: c.origin.x, y: c.origin.y, width: c.maxWidth, height: 48 },
      },
    ];
  }
}
```

يدير المضيف هذه الواجهات عبر سجل واحد، بلا سلسلة شروط `if` لكل نوع. أعد حسابها في كل إطار، وثبّت المفاتيح المسجلة عند مستطيلاتها:

```tsx
const OVERLAYS = { "callout-editor": CalloutEditor };

editor.host.collectOverlays().forEach((o) => {
  const Comp = OVERLAYS[o.key];
  if (Comp) mountAt(o.rect, <Comp overlay={o} editor={editor} />);
});
```

<Callout kind="note" title="يظل المحرك بلا واجهة.">
  تعيد `overlays()` بيانات لا JSX، ولذلك تعمل العقدة نفسها في مضيف React أو
  مضيف DOM عادي أو بلا مضيف على الإطلاق. وهي نقطة الالتقاء التي تنتقل إليها
  النوافذ المنبثقة للصور الأساسية والرياضيات الاختيارية.
</Callout>

<h2 id="mark">تعريف علامة</h2>

العلامة تنسيق مضمّن. تسجّل الدالة المجردة اسم تنسيق صالحًا ومنسوخًا؛ مرّر `render` لرسمه على لوحة الرسم (صنفًا فرعيًا من `Mark`) و`codec` لمنحه محددات Markdown ووسم HTML. إذا حذفت `render` تبقى العلامة عبر CRDT لكنها تُرسم كنص عادي؛ وإذا حذفت `codec` تُسقط عند التصدير إلى Markdown.

```ts
import { defineMark } from "@tasfer/editor";

// minimal — replicates, but paints as plain text and doesn't serialize
const highlight = defineMark("highlight");

// full — on-canvas paint + a `==text==` round-trip
const marked = defineMark("highlight", {
  render: new HighlightMark(), // a Mark subclass; style() returns the paint
  codec: highlightCodec, // delimiters + HTML tag
});
```

يمكن أيضًا لصنف فرعي من `Mark` تعريف `selectionWrap`، وهي المحددات المكتوبة التي تطبق العلامة على تحديد قائم، مثل الإحاطة التلقائية في VS Code مع دلالة Markdown. يؤدي إدخال محرف تشغيل والنص محدد إلى تطبيق العلامة بدلًا من استبدال التحديد (تحدد العلامات الأساسية `` ` `` ← شيفرة و`~` ← يتوسطه خط؛ ويحدد امتداد الرياضيات الاختياري `$` ← معادلة مضمّنة). تتناوب العلامات التي تتشارك محرفًا عند قيم `level` مختلفة حسب عدد المحددات عند تكرار الضغط، مثلما ينتقل التأكيد (`*`، المستوى 1) والغامق (`*`، المستوى 2) من *مائل* ← **غامق** ← ***كلاهما*** ← نص عادي:

```ts
class HighlightMark extends Mark {
  readonly type = "highlight";
  // `=` typed over a selection toggles the highlight (think `==text==`).
  readonly selectionWrap = [{ char: "=" }];
  style() {
    return { background: { color: "#fde68a", padding: 2, borderRadius: 3 } };
  }
}
```

<h2 id="register">تسجيلها</h2>

تعيد `extend` مخططًا *جديدًا* غير قابل للتغيير؛ فلا يتغير المخطط الأساسي أبدًا، ويمكن لمحررين في صفحة واحدة تشغيل مخططين مختلفين.

```ts
import { createEditor, baseSchema } from "@tasfer/editor";

const schema = baseSchema.extend({
  nodes: [callout],
  marks: [highlight],
});

const editor = createEditor({ element, schema });

// The doc carries the data half of the schema, so persistence and markdown
// round-trip the custom block losslessly:
editor.getMarkdown(); // …  <x-callout tone="warn" />  …

// The inferred schema type also reaches every editor operation:
editor.change((c) => {
  c.insertBlock({ type: "callout", tone: "note" });
  c.setBlock({ type: "callout", tone: "warning" });
  // c.setBlock({ type: "calluot" }); // TypeScript error
});

const block = editor.query.block();
if (block?.type === "callout") {
  block.attrs.tone; // inferred from defineNode
}
```

تحمل المواصفة كل جانب يملكه نوعها؛ فلا يوجد سجل منفصل لهذه الجوانب.
يقبل كل من `defineNode` و`defineMark` الخيار `markdownSyntax`، أي متعرّفات
Markdown لصياغة النوع نفسه، ويقبل `defineMark` أيضًا الخيار `structured`
لسلوك المحتوى المنظّم (تهيئة المرفق الفورية، وحل المصدر المرجعي من المرفق،
ومراجع المرفقات، وإعادة كتابة النسخ). لا تخزّن العلامة المنظّمة نص المصدر في
محارف الكتلة؛ بل ترتكز إلى محرف نائب واحد، ويكون مرفقها المرجع الوحيد للمحتوى.
ولا تسري قاعدة صياغة مخصصة إلا عندما تطالب أداة ترميز بالرموز التي تنتجها؛
أما رحلة الذهاب والإياب العامة عبر `<x-type/>` فلا تطالب بأي منها، لذا اقرن
القاعدة بأداة ترميز مكتوبة يدويًا، مثل مواصفات الرياضيات الاختيارية التي تحمل
متعرّفات `$$…$$` و`$…$` بجوار أدوات الترميز التي تقرؤها.

<Callout kind="tip" title="تغييرات المخطط آمنة مع CRDT.">
  تُعنون الكتل باسم *النوع* في السجل المنسوخ، لذلك يدمج نظير يشغّل المخطط نفسه
  تنبيهاتك بلا مشكلة. أما النظير الذي *لا يملك* النوع فلا يتعطل ولا يفقد
  البيانات؛ إذ يحتفظ بالعمليات ويعرض عنصرًا نائبًا «لكتلة غير مدعومة» حتى
  يُحدَّث. انشر تغييرات المخطط إلى جميع الأقران معًا كما تنشر ترحيلًا لقاعدة بيانات.
</Callout>

لتقييد أي من هذه الأنواع المسجلة يمكن لمحرر بعينه *إنشاؤها*، مثل عنوان من سطر
واحد أو صندوق تعليق، ضيّق المخطط باستخدام
[`restrict`](/docs/editor/api-schema#restrict). فهو يضبط الإنشاء وحده؛ ويظل
المحرر المقيّد *يعرض* كتلة `callout` يأتي بها متعاون (أو لصق)، لأن النوع يبقى
مسجلًا.
