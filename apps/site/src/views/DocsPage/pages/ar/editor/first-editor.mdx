<h2 id="mount">الخطوة 1 — تثبيت المحرر</h2>

يحتاج المحرر إلى مضيف ذي ارتفاع، إذ تملأ لوحة الرسم حاويتها. في JavaScript العادي، تمرّر إلى `createEditor` عنصرًا تملكه؛ أما في React فتنشئ `useEditor` المحرر مرة واحدة وتتولى تنظيفه، وتمنحك `containerRef` للتثبيت ومقبض `editor` الحي الذي تعتمد عليه كل الخطوات اللاحقة.

<FrameworkTabs>

```ts file="main.ts"
import { createEditor } from "@tasfer/editor";

// mount into any element with a height: <div id="editor" style="height: 70vh"></div>
const editor = createEditor({
  element: document.querySelector("#editor")!,
  markdown: "# My notes\n\nWrite something.",
  placeholder: "Start writing…",
  autofocus: true,
});
```

```tsx file="App.tsx"
import { useEditor } from "@tasfer/react";

export default function App() {
  const { containerRef, editor } = useEditor({
    markdown: "# My notes\n\nWrite something.",
    placeholder: "Start writing…",
    autofocus: true,
  });

  // the div needs a height — the canvas fills it
  return <div ref={containerRef} style={{ height: "70vh" }} />;
}
```

</FrameworkTabs>

<h2 id="toolbar">الخطوة 2 — شريط أدوات يعكس الحالة</h2>

الفكرة الأساسية هي أن شريط الأدوات *دالة في حالة المحرر*، وليس مكانًا تتتبّع فيه التنسيق بنفسك. اقرأ `activeMarks` لإبراز الأزرار، وأرسل أمرًا عند النقر. في JavaScript العادي، أعد الرسم عند كل `change` و`selectionchange`؛ أما في React فتعيد `useEditorState` عرض شريط الأدوات نيابةً عنك.

<FrameworkTabs>

```ts file="main.ts"
const toolbar = document.querySelector("#toolbar")!;

const buttons = [
  { label: "B", mark: "strong" },
  { label: "i", mark: "emphasis" },
  { label: "</>", mark: "code" },
];

for (const b of buttons) {
  const el = document.createElement("button");
  el.textContent = b.label;
  el.onclick = () => editor.change((c) => c.setMark(b.mark));
  toolbar.append(el);
  b.el = el;
}

// keep the active states in sync — runs on every edit & caret move
function paint() {
  const active = editor.state.activeMarks;
  for (const b of buttons) {
    b.el.classList.toggle("is-active", active.has(b.mark));
  }
}
editor.on("change", paint);
editor.on("selectionchange", paint);
paint();
```

```tsx file="Toolbar.tsx"
import { useEditorState, type TasferEditor } from "@tasfer/react";

const MARKS = [
  { label: "B", mark: "strong" },
  { label: "i", mark: "emphasis" },
  { label: "</>", mark: "code" },
];

export function Toolbar({ editor }: { editor: TasferEditor | null }) {
  const active = useEditorState(editor)?.activeMarks ?? new Set();
  return (
    <div>
      {MARKS.map((b) => (
        <button
          key={b.mark}
          className={active.has(b.mark) ? "is-active" : ""}
          onClick={() => editor?.change((c) => c.setMark(b.mark))}
        >
          {b.label}
        </button>
      ))}
    </div>
  );
}
```

</FrameworkTabs>

في React، اعرض `<Toolbar editor={editor} />` مباشرةً فوق `<div ref={containerRef} />` الخاص بالمحرر؛ وسيظل متزامنًا تلقائيًا من دون استدعاء `paint()` يدويًا.

<Callout kind="tip" title="اختصارات Markdown مفعّلة أصلًا.">اكتب `**bold**`، أو `## ` في بداية السطر، أو `- ` لإنشاء قائمة نقطية؛ فتحوّلها قواعد الإدخال المدمجة أثناء الكتابة. تحيط المحددات أيضًا بالتحديد القائم: تحيط به `(`/`[`/`{`/علامات الاقتباس حرفيًا، بينما يطبق محدد Markdown دلالته؛ يجعل `*` النص مائلًا (واكتبه مجددًا لجعله غامقًا)، وتحوله `` ` `` إلى شيفرة. تبقى علامات الدولار حرفية في هذا الإعداد الأساسي؛ ثبّت الامتداد الاختياري [`mathExtension`](/docs/editor/api-schema#math) عندما يحتاج المنتج إلى معادلة مضمّنة أو معادلة معروضة. يعدّل شريط أدواتك والاختصارات الحالة نفسها.</Callout>

<h2 id="persist">الخطوة 3 — الحفظ محليًا</h2>

احفظ عند التغيير. لأن المستند CRDT، فإن أكثر ما يمكن حفظه موثوقية هو تحديثه الثنائي؛ لكن Markdown مناسب لجهاز واحد ويمكن للبشر قراءته. في React، نفّذ ذلك داخل تأثير يعتمد على `editor`، وأعد دالة إلغاء الاشتراك التي ترجعها `editor.on` لإجراء التنظيف:

<FrameworkTabs>

```ts file="main.ts"
// restore on load
const saved = localStorage.getItem("draft");
if (saved) editor.setMarkdown(saved);

// save on change (debounced by the engine to once per animation frame)
editor.on("change", (tx) => {
  if (tx.isRemote) return;
  localStorage.setItem("draft", editor.getMarkdown());
});
```

```tsx file="App.tsx"
import { useEffect } from "react";

// inside App, after useEditor(...)
useEffect(() => {
  if (!editor) return;

  // restore on load
  const saved = localStorage.getItem("draft");
  if (saved) editor.setMarkdown(saved);

  // save on change (debounced by the engine to once per animation frame)
  return editor.on("change", (tx) => {
    if (tx.isRemote) return;
    localStorage.setItem("draft", editor.getMarkdown());
  });
}, [editor]);
```

</FrameworkTabs>

<h2 id="collab">الخطوة 4 — إضافة التعاون</h2>

هنا تظهر فائدة البدء بـCRDT. لمزامنة نسختين متماثلتين، أرفق مزودًا؛ أي وسيلة نقل ترسل تحديثات CRDT بين الأقران. لا تتغير أي من الخطوات السابقة. افتح الصفحة في علامتي تبويب وشاهد النتيجة. في React، أنشئ المزود داخل تأثير، وتتبّع حالته في الحالة، وأتلفه عند إلغاء التثبيت:

<FrameworkTabs>

```ts file="main.ts"
import { createRelayProvider } from "@tasfer/provider-relay";

const provider = createRelayProvider({
  doc: editor.doc,                 // sync THIS editor's document
  room: "my-notes-demo",           // peers in the same room converge
  relay: "wss://relay.tasfer.app",  // swap for your own; it forwards, can't read
});

// Connection status comes from the provider, not the editor — the provider
// owns the transport, so there is no "sync" event on the editor itself.
provider.on("sync", (s) => {
  status.textContent = s.connected ? `live · ${s.peers} peer(s)` : "offline";
});
```

```tsx file="App.tsx"
import { useState, useEffect } from "react";
import { createRelayProvider } from "@tasfer/provider-relay";

// inside App
const [status, setStatus] = useState("offline");

useEffect(() => {
  if (!editor) return;
  const provider = createRelayProvider({
    doc: editor.doc,                 // sync THIS editor's document
    room: "my-notes-demo",           // peers in the same room converge
    relay: "wss://relay.tasfer.app",  // swap for your own; it forwards, can't read
  });
  // status comes from the provider, not the editor
  provider.on("sync", (s) =>
    setStatus(s.connected ? `live · ${s.peers} peer(s)` : "offline"),
  );
  return () => provider.destroy();   // tear down on unmount
}, [editor]);
```

</FrameworkTabs>

<Callout kind="warn" title="لا يستطيع المُرحِّل قراءة مستندك.">تُشفّر التحديثات من طرف إلى طرف قبل مغادرة علامة التبويب؛ ويمرر المُرحِّل بايتات مبهمة ولا يحتفظ بسجل. يستخدم تطبيق تصفير نموذج المُرحِّل نفسه؛ راجع [إعداد المزامنة والمُرحِّل](/docs/app/sync-relay).</Callout>
