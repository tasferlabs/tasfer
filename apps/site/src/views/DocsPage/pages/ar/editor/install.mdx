<h2 id="install">التثبيت</h2>

أضف الحزمة بمدير الحزم الذي تفضله:

<InstallTabs pkg="@tasfer/editor" />

لا تعتمد الحزمة الأساسية على أي تبعية وقت التشغيل. يأتي CRDT وعارض اللوحة ومُسلسل Markdown داخلها. لا تضف حزمًا أخرى إلا عند الحاجة إلى ربط إطار عمل أو مزود شبكة:

<InstallTabs pkg="@tasfer/react @tasfer/provider-relay" />

<h2 id="requirements">المتطلبات</h2>

- **متصفح يدعم `ResizeObserver` وسياق `<canvas>` ثنائي الأبعاد**، أي كل المتصفحات الحديثة منذ نحو 2020. لا يوجد بديل يعتمد `contenteditable`؛ فاللوحة هي المحرر.
- **ES2020 أو أداة تجميع تستهدفه.** تأتي الحزمة بصيغة ESM مع نقطة توافق CommonJS، والأنواع مضمّنة فلا حاجة إلى `@types`.
- **لا خطوة بناء خاصة بـCRDT.** فهو JavaScript خالص ولا ملفات تنسخها إلى `public/`.

<h2 id="first">محرر في 30 ثانية</h2>

هذا أصغر مثال عملي: مستند قابل للتحرير باختصارات Markdown وسجل تراجع ومؤشر وامض. في JavaScript العادي تثبّت المحرك في عنصر تملكه؛ وفي React تعرض `<Editor>` الذي يملك مضيفه ويضبط حجمه ويرسل التعديلات عبر `onChange`.

<FrameworkTabs>

```ts file="main.ts"
import { createEditor } from "@tasfer/editor";

const editor = createEditor({
  element: document.querySelector("#editor")!,
  markdown: "# Hello\n\nStart typing — *markdown* shortcuts just work.",
  autofocus: true,
});

editor.on("change", () => {
  localStorage.setItem("draft", editor.getMarkdown());
});
```

```tsx file="App.tsx"
import { Editor } from "@tasfer/react";

export default function App() {
  return (
    <Editor
      markdown={"# Hello\n\nStart typing — *markdown* shortcuts just work."}
      autofocus
      style={{ height: "70vh" }}
      onChange={(markdown) => localStorage.setItem("draft", markdown)}
    />
  );
}
```

</FrameworkTabs>
