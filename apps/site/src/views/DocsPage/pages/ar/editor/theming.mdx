<h2 id="tokens">كائن السمة</h2>

مرّر `theme` إلى `createEditor`. توجد الألوان تحت `tokens`؛ وكل حقل اختياري ويُدمج فوق لوحة `DEFAULT_TOKENS` المدمجة، فلا تضبط إلا ما تريد تغييره.

```ts
import { createEditor } from "@tasfer/editor";

const editor = createEditor({
  element,
  theme: {
    // Semantic color tokens — merged over DEFAULT_TOKENS.
    tokens: {
      text: "#1c1c1f",
      heading: "#09090b",
      cursor: "#1db984",
      selection: "rgba(29,185,132,0.30)",
      link: "#1db984",
      codeBackground: "#f4f4f5",
    },
    // The selected font family key + the CSS stacks it can resolve to.
    // (The host is responsible for actually loading the faces.)
    fontFamily: "serif",
    fonts: { families: { serif: '"Libre Baskerville", Georgia, serif' } },
  },
});
```

<Callout kind="note" title="tokens في مقابل styles.">تمثل `tokens` الألوان الدلالية (النص والعنوان والمؤشر والتحديد والرابط و`codeBackground` وغيرها). ولمزيد من التحكم، مثل أحجام الخطوط لكل كتلة وحشوة لوحة الرسم وهندسة شريط التمرير، مرّر كائن `styles` جزئيًا عميقًا يتجاوز أي قيمة منفردة في شجرة الأنماط المحسوبة. تُصدّر `resolveTheme` و`mergeTheme` و`DEFAULT_TOKENS` للمضيفين الذين يريدون حساب النتيجة مسبقًا أو فحصها.</Callout>

<Callout kind="note" title="السمة لكل نوع، ويمكن لكل كتلة تجاوزها.">تمثل `styles.blocks` هنا الطبقة الثابتة على مستوى المستند؛ فهي تنسّق كل كتل النوع بالطريقة نفسها. لتنسيق كتلة واحدة، عيّن حاوية `style` الخاصة بها عبر `setBlock({ style: { … } })` (راجع [الأوامر](/docs/editor/api-commands)). ترتيب التغليب هو: افتراضي العقدة ◁ هذه السمة ◁ `style` الخاص بالكتلة.</Callout>

<h2 id="live">تحديث السمة أثناء التشغيل</h2>

يدمج `editor.setTheme(patch)` رقعةً بعمق في السمة الحالية ويعيد الرسم؛ وهو المسار الوحيد لتغيير السمة أثناء التشغيل، مثل تبديل الوضع الداكن:

```ts
editor.setTheme({
  tokens: { text: "#e6e6e6", heading: "#ffffff", background: "#0b0b0c" },
});
```

<h2 id="css-vars">التحكم بها عبر متغيرات CSS</h2>

لا يقرأ المحرك **DOM مطلقًا**؛ فلا توجد قيمة `"inherit"`. لمتابعة خصائص CSS المخصصة، يقرأ المضيف قيمها ويمررها في `tokens`: مرة عند التثبيت، ومرة أخرى كلما تبدلت فئة السمة.

```ts
function tokensFromCss(el: HTMLElement) {
  const v = getComputedStyle(el);
  return {
    text: v.getPropertyValue("--cy-editor-text").trim(),
    cursor: v.getPropertyValue("--cy-editor-cursor").trim(),
    selection: v.getPropertyValue("--cy-editor-selection").trim(),
  };
}

const editor = createEditor({ element, theme: { tokens: tokensFromCss(element) } });

// Re-read on a dark-mode class flip on <html> and push the new tokens in.
new MutationObserver(() => editor.setTheme({ tokens: tokensFromCss(element) }))
  .observe(document.documentElement, { attributes: true, attributeFilter: ["class"] });
```

<Callout kind="note" title="مطابقة مظهر تصفير.">يمرر مضيف تصفير رموز التصميم `--cy-editor-*` مباشرةً إلى `tokens` بهذه الطريقة بالضبط، لذا لا يحتاج تضمين المحرر ضمن أي سطح من أسطح تصفير إلى لوحة ألوان خاصة به؛ بل يحصل على المتغيرات نفسها التي تستخدمها بقية الواجهة.</Callout>

يُضبط مظهر العنصر النائب لكل نوع كتلة؛ إذ يحمل نمط النص المحسوب لكل كتلة تجاوزًا اختياريًا باسم `placeholder`: اللون `color` (وقيمته الافتراضية `styles.placeholder.color`)، ومقياس الخط `fontScale` (مضاعفًا لحجم خط الكتلة نفسها)، ووزن الخط `fontWeight`.

```ts
editor.setTheme({
  styles: {
    selection: { cornerRadius: 4 },
    blocks: {
      heading1: { placeholder: { color: "#b9b9c2" } },
      quote: { placeholder: { fontScale: 0.8 } }, // a smaller ghost hint
    },
  },
});
```
