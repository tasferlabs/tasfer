<p className="dx-lede">Four things make up the whole surface area: the **value** you put in, the **state** you read out, the **commands** you dispatch, and the **events** you listen to. Learn these and you know the editor.</p>

<h2 id="value">1 — Value in</h2>

An editor is created from markdown, a CRDT document, or nothing at all. These three are equivalent starting points:

```ts
import { createEditor, createDoc } from "@cypherkit/editor";

// from a markdown string
createEditor({ element, markdown: "# Title" });

// from an existing CRDT document (e.g. restored from disk)
const doc = createDoc(savedBytes);
createEditor({ element, doc });

// empty
createEditor({ element });
```

<h2 id="state">2 — State out</h2>

`editor.state` is an immutable snapshot. Read it whenever you need to know what is selected, what marks are active, or render your own UI from it. It never changes underneath you — each edit produces a new snapshot.

```ts
const { selection, activeMarks, doc } = editor.state;

selection.empty;              // true when it's just a caret
activeMarks.has("strong");    // is bold active at the cursor?
```

<h2 id="commands">3 — Changes</h2>

You never mutate the document directly. You call `change(fn)`, which hands you a `ChangeApi`: each method queues a transactional edit (it applies cleanly to the CRDT or no-ops) and chains. The whole callback commits as **one undoable step**, and `change` returns whether anything changed.

```ts
editor.change((c) => c.setMark("strong"));
editor.change((c) => c.setBlock({ type: "heading", level: 2 }));

// several edits, one undo step
editor.change((c) =>
  c.setMark("strong").setMark("emphasis").insertText(" ✶"),
);
```

The full `ChangeApi`, dry runs (`canChange`), and the action bus are on [Changes & actions](/docs/editor/api-commands).

<h2 id="events">4 — Events</h2>

Subscribe to what the editor does. Every listener returns an unsubscribe function.

<PropsTable cols={["Event", "Fires when", "Payload"]} rows={[
  { name: "change", type: "ChangeTransaction", desc: "The document changed — typing, paste, undo, or a remote sync. Payload: { isRemote, ops }." },
  { name: "selectionchange", type: "EditorState", desc: "The caret or selection moved, without a document edit." },
  { name: "focus / blur", type: "EditorState", desc: "The canvas gained or lost focus." },
]} />

Connection status and peer count are **not** editor events — they live on the provider (`provider.on("sync")`), because the provider owns the transport. See [Collaboration](/docs/editor/collaboration).

```ts
const off = editor.on("change", (tx) => {
  if (tx.isRemote) return;          // ignore edits that came from a peer
  console.log("local edit:", tx.ops.length, "ops");
});
// later
off();
```
