<p className="dx-lede">Four things make up the whole surface area: the **value** you put in, the **state** you read out, the **commands** you dispatch, and the **events** you listen to. Learn these and you know the editor.</p>

<h2 id="value">1 ‚Äî Value in</h2>

An editor is created from markdown, a CRDT document, or nothing at all. These three are equivalent starting points:

<FrameworkTabs>

```ts
import { createEditor, createDoc } from "@tasfer/editor";

// from a markdown string
createEditor({ element, markdown: "# Title" });

// from an existing CRDT document (e.g. restored from disk)
const doc = createDoc(savedBytes);
createEditor({ element, doc });

// empty
createEditor({ element });
```

```tsx
import { Editor, useEditor } from "@tasfer/react";
import { createDoc } from "@tasfer/editor";

// from a markdown string
<Editor markdown="# Title" />;

// from an existing CRDT document (e.g. restored from disk)
const doc = createDoc(savedBytes);
<Editor doc={doc} />;

// empty
<Editor />;

// ‚Ä¶or, for full control over the host element, the hook form:
const { containerRef, editor } = useEditor({ markdown: "# Title" });
```

</FrameworkTabs>

<h2 id="state">2 ‚Äî State out</h2>

Read what's selected and which marks are active from an immutable snapshot ‚Äî `editor.state` directly, or `useEditorState(editor)` in React, which returns the same shape and re-renders your component each time it changes. It never changes underneath you; each edit produces a new snapshot.

<FrameworkTabs>

```ts
const { selection, activeMarks, doc } = editor.state;

selection.empty;              // true when it's just a caret
activeMarks.has("strong");    // is bold active at the cursor?
```

```tsx
import { useEditorState } from "@tasfer/react";

const state = useEditorState(editor);   // null until the editor exists
const active = state?.activeMarks ?? new Set();

state?.selection.empty;   // true when it's just a caret
active.has("strong");     // is bold active at the cursor?
```

</FrameworkTabs>

<h2 id="commands">3 ‚Äî Changes</h2>

You never mutate the document directly. You call `change(fn)`, which hands you a `ChangeApi`: each method queues a transactional edit (it applies cleanly to the CRDT or no-ops) and chains. The whole callback commits as **one undoable step**, and `change` returns whether anything changed.

```ts
editor.change((c) => c.setMark("strong"));
editor.change((c) => c.setBlock({ type: "heading", level: 2 }));

// several edits, one undo step
editor.change((c) =>
  c.setMark("strong").setMark("emphasis").insertText(" ‚ú∂"),
);
```

Changes are framework-neutral ‚Äî in React you call `change` on the same handle (the `editor` from `useEditor`, or the one `onReady` gives you), straight from an `onClick`. There's no separate React command API.

The full `ChangeApi`, dry runs (`canChange`), and the action bus are on [Changes & actions](/docs/editor/api-commands).

<h2 id="events">4 ‚Äî Events</h2>

Subscribe to what the editor does. Every listener returns an unsubscribe function.

<PropsTable cols={["Event", "Fires when", "Payload"]} rows={[
  { name: "change", type: "ChangeTransaction", desc: "The document changed ‚Äî typing, paste, undo, or a remote sync. Payload: { isRemote, ops }." },
  { name: "selectionchange", type: "EditorStateSnapshot", desc: "The caret or selection moved, without a document edit." },
  { name: "focus / blur", type: "EditorStateSnapshot", desc: "The canvas gained or lost focus." },
]} />

Connection status and peer count are **not** editor events ‚Äî they live on the provider (`provider.on("sync")`), because the provider owns the transport. See [Collaboration](/docs/editor/collaboration).

Subscribe with `editor.on` directly, or ‚Äî in React ‚Äî from a `useEffect` keyed on the editor, returning the unsubscribe to clean up. For UI that re-renders on edits, prefer `useEditorState` / `useEditorMarkdown` over wiring `on` by hand.

<FrameworkTabs>

```ts
const off = editor.on("change", (tx) => {
  if (tx.isRemote) return;          // ignore edits that came from a peer
  console.log("local edit:", tx.ops.length, "ops");
});
// later
off();
```

```tsx
useEffect(() => {
  if (!editor) return;
  // editor.on returns its unsubscribe fn ‚Äî return it to clean up
  return editor.on("change", (tx) => {
    if (tx.isRemote) return;          // ignore edits that came from a peer
    console.log("local edit:", tx.ops.length, "ops");
  });
}, [editor]);
```

</FrameworkTabs>
