<p className="dx-lede">`@tasfer/react` is the thin React 19 binding over the headless engine ‚Äî a hook to own an editor, a component to render one, and `useSyncExternalStore` subscriptions for toolbars and live preview. Every piece is per-instance, so any number of editors can live on one page.</p>

```ts
import { Editor, useEditor, useEditorState } from "@tasfer/react";
```

<Callout kind="note" title="Options are read once.">Editor options are read at mount only. Reconfigure at runtime through the imperative `TasferEditor` handle (`setTheme`, `setMarkdown`, `change`, ‚Ä¶) ‚Äî not by re-passing props. `<Editor>`'s `onChange` is the one prop that stays live, but it's notify-only: it reports edits, it never pushes content back in. (The document is a CRDT ‚Äî it owns its content, so there is no controlled `value`.)</Callout>

<h2 id="component">&lt;Editor /&gt;</h2>

A drop-in component that renders the host element for you and reports the live editor through `onReady`. It takes every `createEditor` option except `element`, plus the props below ‚Äî including `onChange`, a notify-only edit callback.

<PropsTable rows={[
  { name: "...options", type: "UseEditorOptions", desc: "Every createEditor option except element ‚Äî markdown, doc, schema, theme, placeholder, autofocus, editable." },
  { name: "onChange", type: "(markdown: string) => void", desc: "Notify-only: called with the document's markdown after every change ‚Äî local, paste, undo, or remote. It reports edits but never pushes content back in. For the raw ChangeTransaction (or to ignore remote via tx.isRemote), subscribe through onReady instead." },
  { name: "className", type: "string", desc: "Forwarded to the host <div>." },
  { name: "style", type: "CSSProperties", desc: "Forwarded to the host <div>. The div must be sized ‚Äî give it a height here or via className." },
  { name: "onReady", type: "(editor: TasferEditor) => void", desc: "Called once per editor instance with the live handle. Wire events / persistence here." },
]} />

```tsx
import { Editor } from "@tasfer/react";

// Seed once, react to edits, reach for the handle when you need it.
function App() {
  return (
    <Editor
      markdown="# Hello"
      onChange={save}
      autofocus
      style={{ height: "100vh" }}
      onReady={(editor) => editor.focus("end")}
    />
  );
}
```

<Callout kind="tip" title="Changing content at runtime.">There is no controlled `value` ‚Äî the document is a CRDT and owns its content. To replace it after mount, call `editor.setMarkdown(...)` (or any `editor.change(...)`) on the handle from `onReady`. `onChange` always reports the canonical markdown the editor serialized ‚Äî including remote edits.</Callout>

<h2 id="useeditor">useEditor(options)</h2>

For full control over the host element. Creates the editor exactly once (in a layout effect) and tears it down on unmount. Returns a `containerRef` to attach and the live `editor` (`null` until the first effect runs).

<PropsTable cols={["Returns", "Type", "Description"]} rows={[
  { name: "containerRef", type: "RefObject<HTMLDivElement | null>", desc: "Attach to the host element: <div ref={containerRef} />. The editor fills it, so the div needs a height." },
  { name: "editor", type: "TasferEditor | null", desc: "The live handle, or null on the first render before the mount effect runs." },
]} />

```tsx
import { useEditor } from "@tasfer/react";
import { useEffect } from "react";

function Notes() {
  const { containerRef, editor } = useEditor({ markdown: "# Hello" });

  useEffect(() => {
    if (editor) editor.focus("end");
  }, [editor]);

  return <div ref={containerRef} style={{ height: "100%" }} />;
}
```

<h2 id="state">useEditorState(editor) & useEditorMarkdown(editor)</h2>

`useSyncExternalStore` subscriptions that re-render your component when the editor changes ‚Äî the React-idiomatic way to drive a toolbar or a live markdown preview off editor state.

<PropsTable cols={["Hook", "Returns", "Re-renders on"]} rows={[
  { name: "useEditorState", type: "EditorStateValue | null", desc: "{ selection, activeMarks }. Edits and caret/selection moves. null until the editor exists." },
  { name: "useEditorMarkdown", type: "string", desc: "The serialised document. Every document edit (local or remote)." },
]} />

```tsx
function Toolbar({ editor }: { editor: TasferEditor | null }) {
  const state = useEditorState(editor);
  const active = state?.activeMarks ?? new Set();

  return (
    <button
      data-active={active.has("strong")}
      onClick={() => editor?.change((c) => c.setMark("strong"))}
    >
      Bold
    </button>
  );
}
```

<Callout kind="tip" title="One import surface.">`@tasfer/react` re-exports the engine types you reach for most ‚Äî `TasferEditor`, `Doc`, `ChangeTransaction`, `EditorStateSnapshot`, `CreateEditorOptions` ‚Äî so a consumer can import everything from one place.</Callout>
