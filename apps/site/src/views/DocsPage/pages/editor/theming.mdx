<p className="dx-lede">Because the editor paints its own pixels, theming is done through a plain `EditorTheme` object — semantic color **tokens**, a deep-partial **styles** override, and a **fonts** registry — not CSS selectors into a DOM you don't control.</p>

<h2 id="tokens">The theme object</h2>

Pass a `theme` to `createEditor`. Colors live under `tokens`; every field is optional and merges over the built-in `DEFAULT_TOKENS` palette, so you only set what you change.

```ts
import { createEditor } from "@cypherkit/editor";

const editor = createEditor({
  element,
  theme: {
    // Semantic color tokens — merged over DEFAULT_TOKENS.
    tokens: {
      text: "#1c1c1f",
      heading: "#09090b",
      cursor: "#1db984",
      selection: "rgba(29,185,132,0.30)",
      link: "#1db984",
      codeBackground: "#f4f4f5",
    },
    // The selected font family key + the CSS stacks it can resolve to.
    // (The host is responsible for actually loading the faces.)
    fontFamily: "serif",
    fonts: { families: { serif: '"Libre Baskerville", Georgia, serif' } },
  },
});
```

<Callout kind="note" title="tokens vs styles.">`tokens` are the semantic colors (text, heading, cursor, selection, link, `codeBackground`, …). For finer control — per-block font sizes, canvas padding, scrollbar geometry — pass a deep-partial `styles` object that overrides any individual leaf of the resolved style tree. `resolveTheme`, `mergeTheme`, and `DEFAULT_TOKENS` are exported for hosts that want to precompute or inspect the result.</Callout>

<Callout kind="note" title="Theme is per-type; blocks can override per-instance.">The `styles.blocks` here is the static, document-wide layer — it styles every block of a type the same way. To style one individual block, set its own `style` bag through `setBlock({ style: { … } })` (see [Commands](/docs/editor/api-commands)). The cascade is: node default ◁ this theme ◁ the block's own `style`.</Callout>

<h2 id="live">Updating the theme live</h2>

`editor.setTheme(patch)` deep-merges a patch onto the current theme and repaints — the one path for a runtime theme change, e.g. a dark-mode toggle:

```ts
editor.setTheme({
  tokens: { text: "#e6e6e6", heading: "#ffffff", background: "#0b0b0c" },
});
```

<h2 id="css-vars">Driving it from CSS variables</h2>

The engine **never reads the DOM** — there is no `"inherit"` token. To follow CSS custom properties, the host reads them and forwards the values as `tokens`: once at mount, and again whenever your theme class flips.

```ts
function tokensFromCss(el: HTMLElement) {
  const v = getComputedStyle(el);
  return {
    text: v.getPropertyValue("--cy-editor-text").trim(),
    cursor: v.getPropertyValue("--cy-editor-cursor").trim(),
    selection: v.getPropertyValue("--cy-editor-selection").trim(),
  };
}

const editor = createEditor({ element, theme: { tokens: tokensFromCss(element) } });

// Re-read on a dark-mode class flip on <html> and push the new tokens in.
new MutationObserver(() => editor.setTheme({ tokens: tokensFromCss(element) }))
  .observe(document.documentElement, { attributes: true, attributeFilter: ["class"] });
```

<Callout kind="note" title="Matching the Tasfer look.">Tasfer's host feeds its `--cy-editor-*` design tokens straight into `tokens` exactly this way, so dropping the editor into a Tasfer surface needs no palette of its own — it gets the same variables the rest of the UI uses.</Callout>

<h2 id="metrics">Metrics live in `styles`</h2>

Sizes and spacing aren't tokens — they're leaves of the `styles` tree, each overridable per instance:

```ts
createEditor({
  element,
  theme: {
    styles: {
      blocks: { paragraph: { fontSize: 17 } },
      canvas: { paddingLeft: 64, paddingRight: 64 },
    },
  },
});
```

<h2 id="caret-selection">Caret, selection & placeholder</h2>

The caret, the selection highlight, remote collaborators' carets, and empty-block placeholders are all leaves of `styles` — tune them per instance without touching the DOM.

  { name: "styles.remoteCursor.caretWidth", type: "2", desc: "Width (px) of a peer's caret bar, independent of the local cursor. labelGap (2px) is the gap above the caret to the name flag." },
  { name: "styles.placeholder.showUnfocused", type: "false", desc: "When true, every empty block shows its placeholder — not only the block holding the caret." },
]} />

Placeholder appearance is per block type: each block's resolved text style carries an optional `placeholder` override — `color` (defaults to `styles.placeholder.color`), `fontScale` (a multiple of the block's own font size), and `fontWeight`.

```ts
editor.setTheme({
  styles: {
    selection: { cornerRadius: 4 },
    blocks: {
      heading1: { placeholder: { color: "#b9b9c2" } },
      quote: { placeholder: { fontScale: 0.8 } }, // a smaller ghost hint
    },
  },
});
```
