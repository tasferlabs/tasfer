<h2 id="mount">Step 1 — Mount the editor</h2>

The editor needs a host with a height — the canvas fills its container. In plain JS you hand `createEditor` an element you own; in React, `useEditor` creates the editor once, owns its teardown, and gives you a `containerRef` to mount plus the live `editor` handle every later step builds on.

<FrameworkTabs>

```ts file="main.ts"
import { createEditor } from "@tasfer/editor";

// mount into any element with a height: <div id="editor" style="height: 70vh"></div>
const editor = createEditor({
  element: document.querySelector("#editor")!,
  markdown: "# My notes\n\nWrite something.",
  placeholder: "Start writing…",
  autofocus: true,
});
```

```tsx file="App.tsx"
import { useEditor } from "@tasfer/react";

export default function App() {
  const { containerRef, editor } = useEditor({
    markdown: "# My notes\n\nWrite something.",
    placeholder: "Start writing…",
    autofocus: true,
  });

  // the div needs a height — the canvas fills it
  return <div ref={containerRef} style={{ height: "70vh" }} />;
}
```

</FrameworkTabs>

<h2 id="toolbar">Step 2 — A toolbar that reflects state</h2>

The key idea: the toolbar is a *function of editor state*, never a place where you track formatting yourself. Read `activeMarks` to light up buttons, dispatch a command on click. In plain JS you repaint on every `change` / `selectionchange`; in React, `useEditorState` re-renders the toolbar for you.

<FrameworkTabs>

```ts file="main.ts"
const toolbar = document.querySelector("#toolbar")!;

const buttons = [
  { label: "B", mark: "strong" },
  { label: "i", mark: "emphasis" },
  { label: "</>", mark: "code" },
];

for (const b of buttons) {
  const el = document.createElement("button");
  el.textContent = b.label;
  el.onclick = () => editor.change((c) => c.setMark(b.mark));
  toolbar.append(el);
  b.el = el;
}

// keep the active states in sync — runs on every edit & caret move
function paint() {
  const active = editor.state.activeMarks;
  for (const b of buttons) {
    b.el.classList.toggle("is-active", active.has(b.mark));
  }
}
editor.on("change", paint);
editor.on("selectionchange", paint);
paint();
```

```tsx file="Toolbar.tsx"
import { useEditorState, type TasferEditor } from "@tasfer/react";

const MARKS = [
  { label: "B", mark: "strong" },
  { label: "i", mark: "emphasis" },
  { label: "</>", mark: "code" },
];

export function Toolbar({ editor }: { editor: TasferEditor | null }) {
  const active = useEditorState(editor)?.activeMarks ?? new Set();
  return (
    <div>
      {MARKS.map((b) => (
        <button
          key={b.mark}
          className={active.has(b.mark) ? "is-active" : ""}
          onClick={() => editor?.change((c) => c.setMark(b.mark))}
        >
          {b.label}
        </button>
      ))}
    </div>
  );
}
```

</FrameworkTabs>

In React, render `<Toolbar editor={editor} />` just above the editor's `<div ref={containerRef} />` — it stays in sync automatically, no manual `paint()`.

<Callout kind="tip" title="Markdown shortcuts are already on.">Type `**bold**`, `## ` at the start of a line, or `- ` for a bullet — the built-in input rules transform them as you type. Delimiters also wrap a held selection: `(`/`[`/`{`/quotes enclose it literally, while a markdown delimiter applies its meaning — `*` italicizes (press again for bold) and `` ` `` makes code. Dollar signs stay literal in this core setup; install the opt-in [`mathExtension`](/docs/editor/api-schema#math) when the product needs inline or display math. Your toolbar and the shortcuts edit the same state.</Callout>

<h2 id="persist">Step 3 — Persist locally</h2>

Save on change. Because the document is a CRDT, the most robust thing to persist is its binary update — but for a single device, markdown is fine and human-readable. In React, do it in an effect keyed on `editor` and return `editor.on`'s unsubscribe to clean up:

<FrameworkTabs>

```ts file="main.ts"
// restore on load
const saved = localStorage.getItem("draft");
if (saved) editor.setMarkdown(saved);

// save on change (debounced by the engine to once per animation frame)
editor.on("change", (tx) => {
  if (tx.isRemote) return;
  localStorage.setItem("draft", editor.getMarkdown());
});
```

```tsx file="App.tsx"
import { useEffect } from "react";

// inside App, after useEditor(...)
useEffect(() => {
  if (!editor) return;

  // restore on load
  const saved = localStorage.getItem("draft");
  if (saved) editor.setMarkdown(saved);

  // save on change (debounced by the engine to once per animation frame)
  return editor.on("change", (tx) => {
    if (tx.isRemote) return;
    localStorage.setItem("draft", editor.getMarkdown());
  });
}, [editor]);
```

</FrameworkTabs>

<h2 id="collab">Step 4 — Make it collaborative</h2>

Here's the payoff for being CRDT-first. To sync two replicas, you attach a provider — a transport that ships CRDT updates between peers. Nothing about the steps above changes. Open the page in two tabs and watch. In React, create the provider in an effect, track its status in state, and tear it down on unmount:

<FrameworkTabs>

```ts file="main.ts"
import { createRelayProvider } from "@tasfer/provider-relay";

const provider = createRelayProvider({
  doc: editor.doc,                 // sync THIS editor's document
  room: "my-notes-demo",           // peers in the same room converge
  relay: "wss://relay.tasfer.app",  // swap for your own; it forwards, can't read
});

// Connection status comes from the provider, not the editor — the provider
// owns the transport, so there is no "sync" event on the editor itself.
provider.on("sync", (s) => {
  status.textContent = s.connected ? `live · ${s.peers} peer(s)` : "offline";
});
```

```tsx file="App.tsx"
import { useState, useEffect } from "react";
import { createRelayProvider } from "@tasfer/provider-relay";

// inside App
const [status, setStatus] = useState("offline");

useEffect(() => {
  if (!editor) return;
  const provider = createRelayProvider({
    doc: editor.doc,                 // sync THIS editor's document
    room: "my-notes-demo",           // peers in the same room converge
    relay: "wss://relay.tasfer.app",  // swap for your own; it forwards, can't read
  });
  // status comes from the provider, not the editor
  provider.on("sync", (s) =>
    setStatus(s.connected ? `live · ${s.peers} peer(s)` : "offline"),
  );
  return () => provider.destroy();   // tear down on unmount
}, [editor]);
```

</FrameworkTabs>

<Callout kind="warn" title="The relay can't read your document.">Updates are encrypted end-to-end before they leave the tab; the relay forwards opaque bytes and keeps no log. This is the same relay model the Tasfer app uses — see [Sync & relay setup](/docs/app/sync-relay).</Callout>
