<h2 id="marks">Built-in inline marks</h2>

Marks are inline formatting carried on runs of text. The CRDT names are `strong`/`emphasis`/`strike` (not bold/italic/strikethrough). The togglable ones flip across the selection with `change((c) => c.toggleMark(name))`; `link` and `math` carry data, so they're applied through `setMarkRange` instead.

<h2 id="extend">Extending</h2>

`extend` takes *arrays* of node and mark specs and returns a new, immutable schema — the base is never mutated, so two editors on one page can run different schemas:

```ts
import { baseSchema, defineNode, defineMark } from "@cypherkit/editor";

const schema = baseSchema.extend({
  nodes: [defineNode("callout", { attrs: { tone: { default: "note" } } })],
  marks: [defineMark("highlight")],
});
```

<h2 id="definenode">defineNode(type, config)</h2>

Declares a custom block type. v1 custom nodes are leaf (`content: "none"`) void blocks that round-trip through a generic `<x-type …attrs>` tag, so no tokenizer changes are needed.

<h2 id="definemark">defineMark(type, config?)</h2>

Declares a custom inline mark. With no config it registers a replicated, togglable format name; the optional fields add paint and serialization.

<PropsTable cols={["Field", "Type", "Description"]} rows={[
  { name: "render", type: "Mark", desc: "A Mark subclass instance — its style() returns the on-canvas paint (color, chip, underline). Omit and the mark replicates but paints as plain text." },
  { name: "codec", type: "MarkCodec", desc: "Markdown/HTML round-trip — delimiters (==text==) and an HTML tag. Omit and the mark survives via the CRDT but is dropped on markdown export." },
]} />

```ts
const schema = baseSchema.extend({
  marks: [defineMark("highlight", { render: new HighlightMark(), codec: highlightCodec })],
});
```

Both `defineNode` and `defineMark` return specs you pass to `extend`. See [Custom nodes & marks](/docs/editor/custom-nodes) for the class-first authoring style, node-owned strings, overlay slots, and how the data round-trips through Markdown.
