<p className="dx-lede">`createEditor(options)` returns a `CypherEditor` — the view, the document mutator, and the selection owner over a CRDT document. It's the object you hold onto for the lifetime of the editing surface, and the only handle you need for single-player editing.</p>

<h2 id="create">createEditor(options)</h2>

```ts
import { createEditor } from "@cypherkit/editor";
const editor = createEditor(options: CreateEditorOptions): CypherEditor;
```

<h2 id="reading">Reading state</h2>

The editor never hands you a mutable store. `state` is a fresh, immutable snapshot rebuilt on every read; `getMarkdown()` projects the current document to text. The **`query` facet** is the read mirror of [`change`](/docs/editor/api-commands#targets): every method speaks the **same `DocPoint`/`DocRange` vocabulary the [ChangeApi](/docs/editor/api-commands#targets) accepts** and defaults to the live caret/selection. `block`/`blocks` mirror `change`'s point-vs-range split — `query.block(at)` pairs with `setBlock(attrs, at)`, and `query.blocks(range)` reads the span the selection covers. So you read a position and hand it straight back to a mutation without ever reaching into `getState()`. The selection itself (the caret/anchor `DocRange` with offsets) lives on the reactive snapshot at `state.selection.range`.

<PropsTable cols={["Member", "Type", "Description"]} rows={[
  { name: "state", type: "{ selection, activeMarks, doc }", desc: "Immutable snapshot. selection.range is the current selection as a DocRange (a bare point for a caret), or null; selection.empty is true for a bare caret; activeMarks is a Set of mark names active at the caret. Rebuilt every read." },
  { name: "doc", type: "Doc", desc: "The underlying CRDT document. Pass this to a provider to sync, or call doc.encodeState() to persist." },
  { name: "query.block(at?)", type: "(at?: DocPoint) => BlockData | null", desc: "Plain-data view { id, type, text, attrs } of the single block at at (default: caret block). The point counterpart to query.blocks and the read counterpart to setBlock. Address a block by id with query.block({ block: id }) — the common 'find the block this overlay targets' read." },
  { name: "query.blocks(range?)", type: "(range?: DocRange) => BlockData[]", desc: "The visible blocks the range touches, in document order (default: the selection — equivalently blocks(state.selection.range)). Narrow to a span without fetch-then-filter; for the whole document pass { from: 'start', to: 'end' }." },
  { name: "query.marks(range?)", type: "(range?: DocRange) => Set<string>", desc: "Inline marks active over range (default: selection). With a collapsed range / bare caret, the formats that will apply to text typed there — same set as state.activeMarks; handy for lighting up a toolbar." },
  { name: "getMarkdown()", type: "() => string", desc: "Serialise the current document to markdown." },
  { name: "getState()", type: "() => EditorState | null", desc: "Escape hatch to the raw internal state. Prefer state / the query facet for UI binding." },
  { name: "portalContainer", type: "HTMLDivElement", desc: "Mount point for host overlays (slash menu, link editor) that float over the canvas." },
]} />

<Callout kind="note" title="state is a value, not a store.">`editor.state` is replaced — never mutated — on each transaction. Capture it in a variable only for the duration of one read; for live UI, recompute inside a `change` / `selectionchange` handler (or use `useEditorState` from `@cypherkit/react`).</Callout>

<h2 id="writing">Changing the document</h2>

You never mutate the document directly — you describe edits through the `ChangeApi`, and the editor applies them as CRDT operations. Every method here commits as one undoable step. These are summarised below and documented in full on [Changes & actions](/docs/editor/api-commands).

<PropsTable cols={["Method", "Signature", "Description"]} rows={[
  { name: "change", type: "(fn: (c: ChangeApi) => void) => boolean", desc: "Apply mutations as ONE undoable step. The callback's methods chain. Returns whether anything changed." },
  { name: "canChange", type: "(fn) => boolean", desc: "Dry-run a change: would those mutations change anything right now? (For enabling a toolbar button.)" },
  { name: "setMarkdown", type: "(md: string) => void", desc: "Replace the whole document with markdown. One undoable, broadcast transaction." },
  { name: "undo / redo", type: "() => boolean", desc: "Step local history. Remote edits never enter your undo stack." },
]} />

```ts
editor.change((c) => c.setMark("strong"));

// several edits, one undo step
editor.change((c) =>
  c.setBlock({ type: "heading", level: 1 }).insertText("Untitled"),
);
```

<h2 id="dispatch">Actions & hooks</h2>

Actions are named, dispatchable behaviours a host can observe or override — the seam for slash commands, link clicks, native gestures, and custom mutations. See [Changes & actions](/docs/editor/api-commands#actions) for the three action kinds.

<PropsTable cols={["Method", "Signature", "Description"]} rows={[
  { name: "dispatch", type: "(action, ...args) => boolean", desc: "Dispatch an action through this editor's bus. Returns whether a handler claimed it (or, for a mutation action, whether the document changed)." },
  { name: "registerAction", type: "(action, handler, priority?) => () => void", desc: "Observe or override an action. Higher priority runs first (default 0); return true to claim it. Returns an unsubscribe fn." },
]} />

```ts
import { OPEN_LINK } from "@cypherkit/editor";

// Override a built-in action: open links through the host's own router
// instead of the engine's default. Returning true claims it (stops the
// default); return void to observe and let the default still run.
const off = editor.registerAction(OPEN_LINK, ({ url }) => {
  router.push(url);
  return true;
});

// later — detach the handler
off();
```

<h2 id="facets">View & host facets</h2>

The everyday content/command/query surface lives flat on the handle. Two further groups, kept off the root so the common surface stays small, are reached through namespaced facets — `editor.view.*` for geometry and ephemeral paint, `editor.host.*` for the plumbing you only need when building UI chrome. Both are public and semver-stable; the React bindings never touch them.

<h2 id="events">Events</h2>

`on(event, cb)` subscribes and returns an unsubscribe function. The `change` listener receives a `ChangeTransaction` (`{ isRemote, ops }`); the others receive the internal `EditorState`.

<PropsTable cols={["Event", "Fires when", "Payload"]} rows={[
  { name: "change", type: "ChangeTransaction", desc: "The document changed — typing, paste, undo, or a remote sync. { isRemote, ops }." },
  { name: "selectionchange", type: "EditorState", desc: "The caret or selection moved, with no document edit." },
  { name: "focus / blur", type: "EditorState", desc: "The canvas gained or lost focus." },
]} />

```ts
const off = editor.on("change", (tx) => {
  if (tx.isRemote) return;            // ignore edits that came from a peer
  localStorage.setItem("draft", editor.getMarkdown());
});
// later
off();
```

<Callout kind="note" title="There is no `sync` event on the editor.">Connection status and peer count come from the *provider*, not the editor — the provider owns the transport. Subscribe with `provider.on("sync", …)`. See [Collaboration](/docs/editor/collaboration).</Callout>

<h2 id="lifecycle">Lifecycle</h2>

```ts
const editor = createEditor({ element });

// ... use it ...

// always destroy when the surface unmounts, or you'll leak the canvas
// layers, the ResizeObserver, the portal, and any providers attached to
// editor.doc. createEditor's destroy() does the *full* teardown.
editor.destroy();
```
