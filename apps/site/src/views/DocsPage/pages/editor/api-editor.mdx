<p className="dx-lede">`createEditor(options)` returns a `CypherEditor` ‚Äî the view, the document mutator, and the selection owner over a CRDT document. It's the object you hold onto for the lifetime of the editing surface, and the only handle you need for single-player editing.</p>

<h2 id="create">createEditor(options)</h2>

```ts
import { createEditor } from "@cypherkit/editor";
const editor = createEditor(options: CreateEditorOptions): CypherEditor;
```

<h2 id="reading">Reading state</h2>

The editor never hands you a mutable store. `state` is a fresh, immutable snapshot rebuilt on every read; `getMarkdown()` projects the current document to text. The `getBlock`/`getSelection` family returns plain data in the **same `DocPoint`/`DocRange` vocabulary the [ChangeApi](/docs/editor/api-commands#targets) accepts** ‚Äî so you read a position and hand it straight back to a mutation, without ever reaching into `getState()`.

<PropsTable cols={["Member", "Type", "Description"]} rows={[
  { name: "state", type: "{ selection, activeMarks, doc }", desc: "Immutable snapshot. selection.empty is true for a bare caret; activeMarks is a Set of mark names active at the caret. Rebuilt every read." },
  { name: "doc", type: "Doc", desc: "The underlying CRDT document. Pass this to a provider to sync, or call doc.encodeState() to persist." },
  { name: "getBlock(at?)", type: "(at?: DocPoint) => BlockData | null", desc: "Plain-data view { id, type, text, attrs } of the block at at (default: caret block). The read counterpart to setBlock." },
  { name: "getBlocks()", type: "() => BlockData[]", desc: "All visible blocks, in document order, as plain BlockData." },
  { name: "getSelection()", type: "() => DocRange | null", desc: "The current selection as a DocRange of absolute { block, offset } points (a bare point for a caret), or null when there's no caret. The same currency the ChangeApi accepts." },
  { name: "getMarks(range?)", type: "(range?: DocRange) => Set<string>", desc: "Inline marks active over range (default: selection). getActiveMarks is this applied to the caret." },
  { name: "getActiveMarks()", type: "() => Set<string>", desc: "The inline marks that will apply to text typed at the caret. Same set as state.activeMarks; handy for lighting up a toolbar." },
  { name: "isSelectionEmpty()", type: "() => boolean", desc: "True when there's just a caret (or nothing) ‚Äî no range selected." },
  { name: "getMarkdown()", type: "() => string", desc: "Serialise the current document to markdown." },
  { name: "getState()", type: "() => EditorState | null", desc: "Escape hatch to the raw internal state. Prefer state / the getBlock family for UI binding." },
  { name: "portalContainer", type: "HTMLDivElement", desc: "Mount point for host overlays (slash menu, link editor) that float over the canvas." },
]} />

<Callout kind="note" title="state is a value, not a store.">`editor.state` is replaced ‚Äî never mutated ‚Äî on each transaction. Capture it in a variable only for the duration of one read; for live UI, recompute inside a `change` / `selectionchange` handler (or use `useEditorState` from `@cypherkit/react`).</Callout>

<h2 id="writing">Changing the document</h2>

You never mutate the document directly ‚Äî you describe edits through the `ChangeApi`, and the editor applies them as CRDT operations. Every method here commits as one undoable step. These are summarised below and documented in full on [Changes & actions](/docs/editor/api-commands).

<PropsTable cols={["Method", "Signature", "Description"]} rows={[
  { name: "change", type: "(fn: (c: ChangeApi) => void) => boolean", desc: "Apply mutations as ONE undoable step. The callback's methods chain. Returns whether anything changed." },
  { name: "canChange", type: "(fn) => boolean", desc: "Dry-run a change: would those mutations change anything right now? (For enabling a toolbar button.)" },
  { name: "setMarkdown", type: "(md: string) => void", desc: "Replace the whole document with markdown. One undoable, broadcast transaction." },
  { name: "undo / redo", type: "() => boolean", desc: "Step local history. Remote edits never enter your undo stack." },
]} />

```ts
editor.change((c) => c.setMark("strong"));

// several edits, one undo step
editor.change((c) =>
  c.setBlock({ type: "heading", level: 1 }).insertText("Untitled"),
);
```

<h2 id="dispatch">Actions & hooks</h2>

Actions are named, dispatchable behaviours a host can observe or override ‚Äî the seam for slash commands, link clicks, native gestures, and custom mutations. See [Changes & actions](/docs/editor/api-commands#actions) for the three action kinds.

<PropsTable cols={["Method", "Signature", "Description"]} rows={[
  { name: "dispatch", type: "(action, ...args) => boolean", desc: "Dispatch an action through this editor's bus. Returns whether a handler claimed it (or, for a mutation action, whether the document changed)." },
  { name: "registerAction", type: "(action, handler, priority?) => () => void", desc: "Observe or override an action. Higher priority runs first (default 0); return true to claim it. Returns an unsubscribe fn." },
]} />

```ts
import { OPEN_LINK } from "@cypherkit/editor";

// Override a built-in action: open links through the host's own router
// instead of the engine's default. Returning true claims it (stops the
// default); return void to observe and let the default still run.
const off = editor.registerAction(OPEN_LINK, ({ url }) => {
  router.push(url);
  return true;
});

// later ‚Äî detach the handler
off();
```

<h2 id="events">Events</h2>

`on(event, cb)` subscribes and returns an unsubscribe function. The `change` listener receives a `ChangeTransaction` (`{ isRemote, ops }`); the others receive the internal `EditorState`.

<PropsTable cols={["Event", "Fires when", "Payload"]} rows={[
  { name: "change", type: "ChangeTransaction", desc: "The document changed ‚Äî typing, paste, undo, or a remote sync. { isRemote, ops }." },
  { name: "selectionchange", type: "EditorState", desc: "The caret or selection moved, with no document edit." },
  { name: "focus / blur", type: "EditorState", desc: "The canvas gained or lost focus." },
]} />

```ts
const off = editor.on("change", (tx) => {
  if (tx.isRemote) return;            // ignore edits that came from a peer
  localStorage.setItem("draft", editor.getMarkdown());
});
// later
off();
```

<Callout kind="note" title="There is no `sync` event on the editor.">Connection status and peer count come from the *provider*, not the editor ‚Äî the provider owns the transport. Subscribe with `provider.on("sync", ‚Ä¶)`. See [Collaboration](/docs/editor/collaboration).</Callout>

<h2 id="lifecycle">Lifecycle</h2>

```ts
const editor = createEditor({ element });

// ... use it ...

// always destroy when the surface unmounts, or you'll leak the canvas
// layers, the ResizeObserver, the portal, and any providers attached to
// editor.doc. createEditor's destroy() does the *full* teardown.
editor.destroy();
```
