<h2 id="write">Write with Markdown shortcuts</h2>

Start a new paragraph with one of these patterns. Tasfer converts it as soon as the pattern is complete:

| Type | Result |
| --- | --- |
| `# `, `## `, `### ` | Heading 1, 2, or 3 |
| `> ` | Quote |
| `- `, `* `, `+ ` | Bullet list |
| `1. ` | Numbered list |
| `---` | Divider |
| Three backticks | Code block |
| `$$` | Display equation |

For inline formatting, close the markers around your text:

| Type | Result |
| --- | --- |
| `**bold**` | Bold |
| `*italic*` | Italic |
| `~~strike~~` | Strikethrough |
| `` `code` `` | Inline code |
| `$x^2$` | Inline equation |

You can also use the formatting toolbar and the `/` menu. Markdown is a quick way to write, not a mode you have to stay in.

<h2 id="blocks">Supported Markdown</h2>

Tasfer reads and writes:

- Paragraphs and headings levels 1–3
- Bold, italic, strikethrough, inline code, and links
- Quotes, dividers, bullet lists, numbered lists, and task lists
- Nested lists, using two spaces for each indentation level in exported Markdown
- Fenced code blocks, with an optional language for syntax highlighting
- Images in `![alt](url)` form
- Inline LaTeX between `$...$` and display LaTeX between `$$...$$`

<Callout kind="note" title="Tasfer is not a raw Markdown editor.">
  The editor shows structured, formatted content instead of a source-text pane.
  Markdown is Tasfer's portable input and output format; the live document itself
  is stored as CRDT state so edits can merge safely between devices.
</Callout>

<h2 id="import-export">Import and export</h2>

Import `.md` files from **Settings → Data → Import**. You can import individual Markdown files or a ZIP containing them.

Export one page as Markdown from its page menu. To back up every page at once, use **Settings → Data → Export all**; Tasfer creates a ZIP of plain `.md` files. The same page menu can also export HTML, PDF, or plain text.

<Callout kind="tip" title="Keep a portable copy.">
  Markdown export is a one-way copy you can open in another editor, keep in Git,
  or use as a backup. Tasfer's local database remains the source of truth for
  editing and peer-to-peer sync.
</Callout>
