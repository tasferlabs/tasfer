<h2 id="first-doc">Your first document</h2>

<Steps>
  <Step title="Create a space">
    On first launch, Tasfer asks you to create a space — or join someone else's with an invite. A space is a collection of pages that lives entirely on your device, in a local database.
  </Step>
  <Step title="Write">
    Markdown shortcuts work as you type: `#` for headings, `**bold**`, `>` for quotes, `- [ ]` for tasks. Everything is saved locally the instant you type — there is no "save" button, but all your changes are backed with with you.
  </Step>
  <Step title="Export any time">
    **Settings → Data → Export all** writes every page out as plain markdown in a ZIP — files you can open in any editor, today or in twenty years. Individual pages export as Markdown, HTML, PDF, or plain text, and the same Data tab imports markdown files or a ZIP back in.
  </Step>
</Steps>

<Callout kind="note" title="No telemetry, ever.">
  Tasfer ships with no analytics SDK, no crash reporting, and no account system.
  The network is used for two things only: syncing spaces you have shared, and
  (on desktop) checking GitHub for app updates. Your documents are never
  uploaded anywhere. See [Privacy & data](/docs/app/privacy) for the complete
  accounting.

  Because nothing phones home, the only way we hear about a bug is if someone
  reports one. Hit something? [Open an issue on GitHub](https://github.com/hamza512b/tasfer/issues).
</Callout>
