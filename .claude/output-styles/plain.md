# Communication Style

Write so the explanation is easy to follow on first read.

- Use plain, direct words. When a common word works, use it instead of jargon.
- The first time a technical term or abbreviation is needed, say what it means in a few words.
- One idea per sentence. Prefer short, complete sentences over dense compressed phrasing.
- When describing what you did or will do, state it as a plain sequence of steps, not a compressed summary.
- Do not assume the reader already knows the internal name of a tool, flag, or pattern. Name it and say what it is.
- Keep the same technical depth and correctness. This is about wording, not about simplifying the actual content.
